Infrastructure

OTLCBot runs entirely on a Raspberry Pi Zero W — a single-board computer with 512 MB of RAM and a single-core 1 GHz processor. The system uses systemd for process management, cron for scheduled tasks, SQLite for storage, and a Cloudflare tunnel for external access to the dashboard.

Services

ServiceUnit fileProcessPurpose
otlcbototlcbot.servicepython src/bot.pyTelegram bot: message logging, commands, agent triggering
otlcbot-webappotlcbot-webapp.servicepython src/webapp.pyFlask dashboard API on port 5000
cloudflared-tunnelcloudflared-tunnel.servicescripts/start-tunnel.shCloudflare tunnel with auto-URL capture

All three services are enabled at boot (WantedBy=multi-user.target) and configured to restart on failure. The tunnel service depends on the webapp service (Requires=otlcbot-webapp.service).

Database Schema

All data lives in a single SQLite file (default: /opt/otlc/data.db). Tables are created by init_db() in bot.py on startup, with migrations handled via ALTER TABLE wrapped in try/except blocks (SQLite doesn't support IF NOT EXISTS for columns).

Database Maintenance

Backup

scripts/backup_db.sh copies the database file to a timestamped backup. Runs weekly via cron.

Pruning

src/prune_db.py deletes messages older than RETENTION_DAYS (default: 365). Optionally runs VACUUM to reclaim disk space. Uses a repo-relative path to find .env, so it works correctly from cron.
