OTLCBot

Green links(w) point to Wikipedia articles on AI development concepts.

OTLCBot is a Telegram bot built on a Raspberry Pi Zero W that performs long-term observation of group chat activity. It uses a large language model(w) (Google Gemini) to generate weekly reports, maintain rolling personality profiles of its subjects, and power an autonomous agent(w) that occasionally interjects with dry, observational commentary. The bot's behavior is shaped by a system prompt(w) describing it as a not-quite-human intelligence conducting field research on a group of friends — a persona inspired by David Foster Wallace's critique of irony and sincerity. It has been in continuous operation since early 2026.

Overview

OTLCBot was built as a personal project to add a layer of structured observation to a small network of friend-group Telegram chats. It combines message logging, social features (group betting, shared watchlists), and AI-generated content through Google Gemini into a system that watches, remembers, and — on occasion — speaks. The project demonstrates several practical patterns in modern AI application development, including retrieval-augmented generation(w), agentic reasoning, and multimodal generation (text and images from a single API).

The bot monitors two deployment contexts. The first is a standalone group chat. The second is Owl Town, a constellation of six topic-specific chats (books, sports, health, AI, media, and a general "basement") whose activity is aggregated into a single combined weekly report. All AI-generated content passes through a consistent persona that treats the group members as subjects in an ongoing study. See Architecture for the full system design.

Agent System

The bot includes an agentic decision layer — a system where the LLM does not simply respond to user input but autonomously decides whether and how to act. This follows an Observe–Reason–Act loop: after a configurable number of messages (default: 30), the agent gathers context, passes it to Gemini, and receives back a JSON decision — an example of structured output(w) where the model is constrained to respond in a machine-readable format. Most of the time, it chooses silence. See Agent System for the full architecture.

The agent has six tools(w) at its disposal — discrete functions the LLM can invoke by name, a pattern sometimes called function calling:

ToolWhat it does
send_commentaryPosts a brief field observation about recent conversation
illustrated_summaryGenerates an AI cartoon of the week's activity with a caption
nudge_betReminds the group about a stale, unsettled bet
sincerity_checkRuns the DFW Sincerity Index and shares group-level results
add_mediaExtracts an organic media recommendation from chat and adds it to the watchlist
update_casefileIdentifies a personality-revealing moment, updates a subject's profile, and announces the discovery

Tools self-register via a Python decorator pattern — each tool function is wrapped with metadata (name, description, cost, guidelines) that automatically adds it to a tool registry. The agent's system prompt is generated dynamically from this registry, so adding a new tool requires no changes to the reasoning logic. The agent enforces a 2-hour cooldown between actions, ignores quiet chats, and is prohibited from focusing on the admin user.

Weekly Reports

Every Friday at 3pm EST, a cron job triggers the full weekly reporting pipeline. For each monitored chat, the bot assembles a report containing message counts, the most active participants, and an AI-generated recap. Rather than sending random message samples to the LLM, the system uses conversation burst detection to select coherent conversation windows. This is a form of retrieval-augmented generation (RAG)(w): the system retrieves relevant context from its own database and injects it into the prompt at generation time. See Weekly Reports for the complete pipeline.

The report includes a DFW Sincerity Index: a Gemini-powered sentiment analysis(w) that assesses the group's irony-to-sincerity ratio, scored as a percentage and converted to a letter grade. Each user receives a private DM with their individual score and a trend arrow. The group sees only the aggregate.

For Owl Town, the bot produces a combined report accompanied by a prose gazette — a ~200-word briefing summarizing the week as a field report. A weekly AI-generated cartoon is also produced using Gemini's multimodal image generation.

Memory System

One of the core challenges in building LLM-powered applications is memory — language models are stateless by default and have a limited context window(w). OTLCBot addresses this with a three-tier memory architecture. See Memory System for details.

Tier 1 — Case Notes (short-term). Raw observations recorded by the agent as it acts during the week, tagged by type and optional target user.

Tier 2 — Profiles and Themes (long-term). Every Friday, Gemini merges each user's raw messages with Tier 1 notes into a rolling personality summary. The prompts instruct the model to "consolidate and merge — don't just append," making this a form of lossy compression that fits an ever-growing observation history into a fixed-size context window.

Tier 3 — Execution Log. Records every agent decision for cooldown enforcement and debugging.

The key design is the upward flow: Tier 1 notes are injected into Tier 2 consolidation prompts each Friday. The agent also reads its last five notes before each decision, giving it continuity between firings. When the system needs current state, it assembles a grounding block — a technique that reduces hallucination(w) by giving the model verified data rather than asking it to guess.

Dashboard

The /dashboard command opens a Telegram MiniApp — a single-page web application served by a Flask backend and exposed via a Cloudflare tunnel. It provides five tabs: Gallery (weekly AI cartoons), User Profiles (case file dossiers), Bets (open and settled), Watch/Read (shared media list with star ratings), and Field Notes (a timeline of the bot's observations). See Dashboard for the technical implementation.

Infrastructure

OTLCBot runs on a Raspberry Pi Zero W with three systemd services: the Telegram bot, the Flask webapp, and a Cloudflare tunnel manager. Three cron jobs handle the Friday weekly report, a weekly database backup, and a monthly message prune. All data lives in a single SQLite database with ten tables. Configuration is .env-driven with toggleable feature flags. See Infrastructure for the full setup.

Personality

The bot's voice is defined by a single BOT_PERSONA constant — a system prompt injected into every Gemini call. System prompts are a foundational technique in LLM development: persistent instructions prepended to every request that shape the model's tone and role. The persona describes the bot as an AI quietly observing humans it finds confusing, sentimental, and — in Wallace's words — "unavoidably naive and goo-prone."

Different LLM calls use different temperature(w) settings — a parameter controlling randomness in output. Analytical tasks use low temperature (0.3–0.5) for deterministic results; creative tasks use high temperature (1.2–1.5) for varied prose. This dual-temperature approach is common in production LLM systems that need both reliable structured output and expressive natural language.
