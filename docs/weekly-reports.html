Weekly Reports

Every Friday at 3pm EST, a cron job runs src/weekly.py, which executes the full reporting pipeline. This is the most Gemini-intensive operation in the system, typically making 10–20 API calls per run. The pipeline handles per-chat reports, sincerity analysis, image generation, user profile updates, and the Owl Town combined report.

Pipeline Sequence

For each monitored chat (except those covered by the Owl Town combined report):

Conversation Burst Detection

Rather than sampling random messages (which tends to produce incoherent, context-free snippets), the system identifies conversation bursts — clusters of messages sent within a short time window of each other. This is a practical application of RAG(w), where the retrieval step is designed to preserve conversational coherence.

The algorithm (_detect_bursts()) works by scanning messages chronologically and splitting into groups whenever the gap between consecutive messages exceeds a threshold (default: 10 minutes). Bursts smaller than 3 messages are discarded. The remaining bursts are formatted with usernames and timestamps, then truncated to fit within a character budget. This produces representative conversation excerpts that the LLM can actually understand in context.

DFW Sincerity Index

The sincerity index is a sentiment analysis(w) system themed around David Foster Wallace's writing on irony and sincerity. Gemini analyzes a sample of the week's messages and returns a JSON object with an irony percentage for each user and the group as a whole.

Trend Tracking

Scores are saved to the sincerity_scores table each week. The bot compares the current week's score to the previous week and shows a trend arrow in the DM: rising irony, falling irony, or stable. The group message shows only the aggregate grade and trend — individual scores are private.

Image Generation

Weekly cartoons are generated in two steps, a common pattern when using multimodal LLMs:

Step 1: Scene description. Gemini (gemini-2.5-flash-lite) reads the conversation windows and group theme, then writes a short scene description suitable for image generation — essentially translating chat content into a visual concept.

Step 2: Image generation. Gemini (gemini-2.5-flash-image) generates the cartoon from the scene description. The result is sent to the chat and the Telegram file ID is saved for the dashboard gallery.

Owl Town Combined Report

After individual chat reports are sent, the pipeline aggregates all Owl Town sub-chats into a single combined report. This uses get_conversation_windows_multi(), which pulls burst-detected conversation windows from every sub-chat and labels them by channel name.

The combined report includes a per-channel message breakdown, top posters across all channels, a combined AI recap, and a combined sincerity index.

The Gazette

In addition to the structured report, the bot generates a prose gazette via build_weekly_gazette(). This takes the assembled statistics, sincerity scores, and any case notes from the week and asks Gemini to write a ~200-word field report in the bot's observational voice. The gazette is sent as the lead message before the structured stats, giving the Owl Town report a more narrative feel.

Admin Cost DM

After all reports are sent, the bot DMs the admin user with a cost summary. This includes the number of images generated, the number of text API calls, the estimated cost for this run, the projected monthly cost, and Pi system health metrics (CPU temperature, memory usage, disk space, load average, uptime). Cost tracking is important in LLM-powered applications where usage-based pricing can be unpredictable.
