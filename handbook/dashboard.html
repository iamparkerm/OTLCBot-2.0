Dashboard

The OTLCBot dashboard is a Telegram MiniApp — a web application that opens inline within the Telegram client when a user taps the /dashboard command. It provides a browsable interface to the bot's generated content, profiles, bets, watchlist, and field notes.

Architecture

The dashboard has three layers:

LayerTechnologyFile
FrontendVanilla HTML/CSS/JS (single-page app)src/gallery.html
BackendFlask REST APIsrc/webapp.py
TunnelCloudflare quick tunnelscripts/start-tunnel.sh

The frontend is a single HTML file with no build step, no framework, and no external dependencies beyond the Telegram WebApp SDK. This is a deliberate choice for a Pi Zero W deployment where simplicity and minimal resource usage matter.

Telegram WebApp Integration

When the user taps /dashboard, the bot sends an inline keyboard button with a WebApp URL. Telegram opens this URL inside the app as a MiniApp panel. The URL includes the chat_id as a query parameter so the dashboard knows which chat's data to display.

The frontend accesses window.Telegram.WebApp to read the current user's identity (tg.initDataUnsafe.user), which is used for features like watchlist ratings — the app knows who is rating without requiring a separate login. It also reads Telegram's theme variables (--tg-theme-bg-color, --tg-theme-text-color, etc.) so the dashboard automatically matches the user's light or dark mode.

Tabs

Gallery

Displays all AI-generated weekly cartoons for the chat, newest first. Each card shows the image, the week date, and the image prompt used to generate it. Images are lazy-loaded and proxied through the backend (/img/<id>) to keep the bot token server-side. This is the default tab, loaded on page open.

User Profiles

Shows case file dossiers for all users active in the chat who have a generated profile. Each card displays the username, the case file text in monospace font, the profile version number, and the last update date. Profiles are loaded lazily on first tab click.

Bets

Lists all bets (open and settled) for the chat, newest first. Open bets have a green left border; settled bets are dimmed with the winner shown. Each card displays the bet description, settlement criteria, wager, creator, and creation date.

Watch/Read

A shared media tracking list. Users can add items via an inline form (title + media type) or via the /watch and /read chat commands. Each item shows its title, type icon, who added it, and when. Users can rate items 1–5 stars; the average rating and count are displayed. Ratings are per-user and can be updated by clicking a different star.

Field Notes

A timeline of the bot's observations from the case_notes table. Each note card is color-coded by type: green for commentary, purple for discoveries (with the target username shown), and blue for general observations. Notes are displayed newest first with the creation date.

Signed Integer URL Converter

Telegram chat IDs are negative integers (e.g., -1001320128437). Flask's default integer URL converter doesn't handle negative numbers, so the app registers a custom SignedIntConverter that accepts the -?\d+ pattern.

Image Proxy and Caching

Weekly images are stored in Telegram's servers, referenced by a file_id. To display them in the webapp without exposing the bot token, the backend resolves each file_id to a temporary file_path via the Telegram API, then streams the image. File paths are cached in-memory for 1 hour (Telegram's validity window).

Cloudflare Tunnel

The webapp runs on the Pi's local network (port 5000) and is exposed to the internet via a Cloudflare quick tunnel. A systemd service (cloudflared-tunnel.service) manages the tunnel lifecycle: on start, it launches cloudflared, captures the generated URL from stdout, updates WEBAPP_URL in .env, and restarts the bot so it picks up the new URL. This makes the tunnel self-healing — if the Pi reboots or the tunnel crashes, a new URL is automatically provisioned and configured.
